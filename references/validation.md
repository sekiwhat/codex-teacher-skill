# Validation Rules

交付前必须完成本文件所有检查。验证失败必须修复并重新检查，不得直接交付。

## 公式与结构验收

- KaTeX `delimiters` 必须同时包含 `$$...$$`、`$...$`、`\(...\)`、`\[...\]`，并与正文实际定界符一致。
- `.math-block` 中的展示公式必须使用 `$$...$$` 或 `\[...\]`；生成后扫描全部 `.math-block`，不得残留单美元展示公式。
- 若用 JavaScript `String.replace` 生成含 `$` 的 HTML，必须使用函数替换器或正确转义；替换字符串中的 `$$`、`$&`、`$'` 等具有特殊语义，禁止未经验证直接写入。
- 确认文件只有一个 `<!DOCTYPE html>`、一个 `<html>` 根节点和一个 `</html>`，不得重复拼接整份文档。
- 检查标签闭合、公式中的 `<`/`>` 转义以及 KaTeX 初始化脚本完整。
- 浏览器可用时刷新并确认目标公式生成 `.katex` 或 `.katex-display`；不可用时至少完成静态扫描。

## 公式字符转义

- HTML 源码里的公式中，裸 `<` 和 `>` 必须转义为 `&lt;` 和 `&gt;`。
- 浏览器 HTML 解析器会把裸 `<` 当作标签开头吞掉后续内容，导致公式被截断、KaTeX 渲染失败。
- 示例：`$i<j$` 必须写成 `$i&lt;j$`，`$0<x<1$` 必须写成 `$0&lt;x&lt;1$`。
- 这是生成时在 HTML 源码中转义，不是运行时 JS 处理。不要用 DOM `nodeValue` 赋值来补救，因为 KaTeX 会收到字面的 `&lt;` 而不是 `<`。

## 高亮器验收

- 最终 HTML 必须完整内联 `assets/highlighter.css`、`assets/highlighter-toolbar.html`、`assets/highlighter.js`。
- 页面必须支持自定义颜色、修改、删除、全部清除和 `localStorage` 持久保存。
- 如果生成或重做 HTML，必须重新读取三份高亮器资产，不得凭记忆重写。

## 源代码块转义验收

- `<pre><code>` 中的源码必须保留原始转义字符，尤其是 C/C++/Java/JavaScript 字符串里的 `\n`、`\t`、`\\`、`\"` 等。
- 生成 HTML 时不得让生成脚本语言把这些转义序列解释成真实换行、制表符或其他字符。
- 若用 Python、JavaScript 或其他脚本生成 HTML，源码片段必须使用原始字符串、双重转义，或先逐字读入原文再做 HTML 转义。
- 禁止把含 `\n` 的源码放进会解释转义序列的普通字符串字面量。

源码 `write(fd, "world\n", 6);` 在最终 HTML 的代码块中必须显示为反斜杠加 `n`，不得显示成字符串内部真实换行：

```c
write(fd, "world
", 6);
```

交付前必须扫描全部 `<pre><code>` 代码块。任何字符串字面量内部含真实换行都视为验证失败，必须修复并重新检查；代码输出示例块不属于此检查，但源码块必须检查。

## 最终静态检查清单

- 只有一个 `<!DOCTYPE html>`。
- 只有一个 `<html` 开始标签和一个 `</html>` 结束标签。
- KaTeX CSS、KaTeX JS、auto-render JS 和初始化脚本存在。
- 四类公式定界符都在初始化脚本中。
- `.math-block` 内没有单美元展示公式。
- 公式中的 `<` 和 `>` 已在 HTML 源码层面转义。
- 高亮器 CSS、toolbar HTML 和 JS 已内联。
- `<pre><code>` 源码块中的 `\n`、`\t`、`\\` 没有被错误解释。
