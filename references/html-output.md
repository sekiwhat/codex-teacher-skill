# HTML Output Rules

## 格式来源

- `HTML_FORMAT.md` 是默认 HTML 视觉格式、页面骨架、CSS 组件和排版规则的唯一格式源。
- 除非用户本次明确指定另一种参考格式，否则必须按 `HTML_FORMAT.md` 创建 HTML。
- 如果旧说明、记忆或示例与 `HTML_FORMAT.md` 冲突，格式相关要求以 `HTML_FORMAT.md` 为准；内容完整性、语言配对和公式安全要求仍以本 skill 的引用文件为准。
- 每次生成 HTML 时必须读取并内联 `assets/highlighter.css`、`assets/highlighter-toolbar.html`、`assets/highlighter.js`，为页面提供自定义颜色、可持久保存的文字高亮功能。

## 文件命名与位置

- 文件名：`teaching_[原文件名].html`。例如 `chapter1.md` 输出为 `teaching_chapter1.html`。
- 默认位置：当前项目文件夹或工作区根目录，即本次 Codex 运行的 `cwd`，不要默认写入原 Markdown 文件同目录。
- 用户指定优先：若用户本次明确指定输出目录或完整输出路径，则按用户指定位置写入。
- 若无法确定项目根目录或没有写入权限，先询问用户确认输出位置。

## HTML 语言模式

- 英文模式：`<html lang="en">`
- 中文模式：`<html lang="zh-CN">`
- 中英文对照模式：`<html lang="zh-CN">`，并遵守 `references/language-modes.md` 的原子配对规则。

## 内容结构

在 `HTML_FORMAT.md` 的页面骨架内组织内容，至少包含：

1. 文档标题和教学讲解标题。
2. 左侧 sticky 目录栏；移动端为可展开的 `Table of Contents 目录` 按钮。
3. 知识地图，列出全部知识点。
4. 按原文档顺序展开的章节。
5. 定义、例子、例题、证明、注意、总结等教学组件。
6. 全文总结，包括知识脉络回顾和核心要点清单。

组件的具体类名、配色、封面、正文容器、目录、章节、知识框、响应式和打印样式统一以 `HTML_FORMAT.md` 为准。

## 数学公式和离线降级

- 默认使用 KaTeX CDN 渲染数学公式。
- 生成 HTML 时始终包含 KaTeX CDN 引用和初始化脚本。
- 如果用户反馈公式未渲染，提供降级版本：移除 KaTeX 引用，公式用 `<code>` 标签包裹为纯文本，并在页面顶部提示“公式以纯文本形式展示”。
- 数学公式使用 KaTeX 语法：行内 `$...$`，独立 `$$...$$`，也支持 `\(...\)` 与 `\[...\]`。

## 完成后回复

生成并验证 HTML 后，回复用户：

1. 教学文件已生成。
2. 绝对文件路径。
3. 提示用户可以在浏览器中打开查看。
4. 不超过 5 行的内容概要，说明覆盖了哪些知识点。
