# Teacher Workflow

## Checkpoint 0：确认语言

检查用户在本次请求中是否已经明确指定英文、中文或中英文对照。

- 若未指定，立即询问：“教学 HTML 希望使用英文、中文，还是中英文对照？”
- 等待用户回答后再读取和处理文档。
- 不得依据用户的界面语言、文档语言或历史调用自行推断。
- 若用户已经明确指定，则不重复询问。

## 详细步骤

1. 通读全文：完整阅读整个 Markdown 文档，理解全貌和结构。
2. 识别知识点：列出文档中涉及的所有概念、定义、定理、公式、例题、证明、代码和重要结论。
3. Checkpoint 1：展示知识点清单，询问："以上是本文档涉及的 N 个知识点，是否完整？需要补充或调整吗？" 等用户确认后再继续。
4. 逐一讲解：按照文档顺序，对每个知识点进行详尽讲解，并严格使用 Checkpoint 0 确认的语言模式。
5. 串联总结：给出整体知识脉络总结，帮助用户看到知识之间的关系。
6. Checkpoint 2：展示 HTML 大纲结构（目录和各章节标题），询问："将按此结构生成 HTML，是否需要调整？" 等用户确认后再继续。
7. 读取格式并输出 HTML：完整读取 `HTML_FORMAT.md`、`references/html-output.md`、`references/validation.md` 和三份 `assets/highlighter.*` 标准资产；按页面骨架、设计系统和组件映射生成单个 HTML 文件。
8. 强制验证：写入后扫描全部公式、HTML 结构、高亮器资产和源码代码块；验证失败必须修复并重新检查，不得直接交付。

## 大文档处理

如果文档超过 500 行，先询问用户是否分章节生成多个 HTML。用户选择一次性生成时继续；用户选择分章节时按原文档一级标题拆分。
